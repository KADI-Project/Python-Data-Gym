Introduction to Python Classes

A quick 10-minute overview of object-oriented programming with Python
✅ Focus: Classes, inheritance, and super().__init__()

What Is a Class?

Classes are blueprints for creating objects (instances).

class Instrument:
    def __init__(self, model, serial_number, location):
        self.model = model
        self.serial_number = serial_number
        self.location = location
  • Defines data (attributes) and behavior (methods)
  • Creates reusable templates for real-world entities

Creating and Using Objects

location = Location("2024-01-01", 47.0, 8.0, 500)
instr = Instrument("Basic-1000", "SN123", location)
print(instr.model, instr.location.latitude)
  • instr is an instance of Instrument
  • __init__() initializes attributes
  • location is an instance of another class: Location

Supporting Class: Location

class Location:
    def __init__(self, date, latitude, longitude, distance_from_reference_surface):
        self.date = date
        self.latitude = latitude
        self.longitude = longitude
        self.distance_from_reference_surface = distance_from_reference_surface
  • Used to describe where the instrument is deployed
  • Promotes code reuse and clean design

Inheritance Basics

class Instrument:
    def __init__(self, model, serial_number, location):
        self.model = model
        self.serial_number = serial_number
        self.location = location

    def measure(self):
        raise NotImplementedError

    def flag_measurement(self):
        raise NotImplementedError

class Thermometer(Instrument):
    def __init__(self, model, serial_number, location, unit):
        self.unit = unit

❗ Issue: Instrument.__init__() not called ➔ base attributes not initialized

Fix with super().__init__()

class Thermometer(Instrument):
    def __init__(self, model, serial_number, location, unit):
        super().__init__(model, serial_number, location)
        self.unit = unit
  • super().__init__() calls parent class's constructor
  • Ensures Instrument sets up shared attributes

Full Example with Override

class Thermometer(Instrument):
    def __init__(self, model, serial_number, location, unit):
        super().__init__(model, serial_number, location)
        self.unit = unit

    def measure(self):
        return 20.5

    def flag_measurement(self):
        print("Measurement verified.")

location = Location("2024-01-01", 47.0, 8.0, 500)
t = Thermometer("Thermo-X", "SN456", location, "C")
print(t.measure(), t.unit)

Instrument Methods: Real-World Meaning

class Thermometer:
    def measure(self):
        """Real-world equivalent: Send a command to the instrument,
        wait for response, and return the reading."""
        raise NotImplementedError

    def flag_measurement(self):
        """Real-world equivalent: Check if measurement falls within
        expected physical limits or QA/QC thresholds and return a quality flag (e.g., 0 = good, 1 = questionable, 2 = bad)."""
        raise NotImplementedError
  • measure() could return a sensor reading
  • flag_measurement() returns a quality flag based on whether the value is within expected physical limits

Additional Class: Operator

class Operator:
    def __init__(self, name, organization, email, from_date, to_date):
        self.name = name
        self.organization = organization
        self.email = email
        self.responsible_from_date = from_date
        self.responsible_to_date = to_date
  • Represents a person or organization operating the instrument
  • Can be associated with multiple instruments

Why Use Classes?

  • Group data + behavior logically
  • Reuse code via inheritance
  • Improve structure and clarity
  • Enable abstraction, encapsulation, polymorphism
  • Model real-world entities like instruments, locations, and people

Recap & Resources

  • Classes define object structure and behavior
  • __init__() initializes an instance
  • super().__init__() lets you extend base classes
  • Use composition (e.g., Location) and inheritance (e.g., Thermometer)
